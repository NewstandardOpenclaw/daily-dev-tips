Laravel（今日のサブトピック: Eloquent N+1対策 / 複合Index設計）
  • 概要: 一覧APIの実行速度を、Eloquentの取得戦略とDB indexで同時に改善する。
  • なぜ面白いか: アプリ層とDB層の改善が直結し、体感速度が大きく変わる。
  • 現場での使いどころ: 管理画面一覧や検索APIの応答が重いときの即効改善。
  • ハマりどころ: with()でN+1を消しても、where/orderBy列のindex不足で遅いまま。
  • 基礎の復習: belongsTo/hasManyの向き、必要列だけselect、paginateで負荷分散。
  • 今日の実践ミニ課題: 既存一覧1本でクエリ数・応答時間・EXPLAINをbefore/after比較。
$posts = Post::query()
  ->with(['user:id,name'])
  ->where('status', 'published')
  ->orderByDesc('published_at')
  ->paginate(20);

TypeScript（Day: 型ガードでunknownを安全に扱う）
  • 概要: unknown/union型を型ガードで絞り込み、安全な分岐を作る。
  • なぜ面白いか: ランタイムチェックがそのまま型安全につながり、事故を減らせる。
  • 現場での使いどころ: APIレスポンス検証、フォーム入力の動的分岐処理。
  • ハマりどころ: asで押し切る実装は短期的に楽でも保守で破綻しやすい。
  • 基礎の復習: type predicate（v is X）とin演算子の使いどころを整理。
  • 今日の実践ミニ課題: 既存のas anyを1箇所、型ガード関数へ置換してテスト実行。
function isUser(v: unknown): v is { id: number; name: string } {
  return typeof v === 'object' && v !== null && 'id' in v && 'name' in v;
}

参考: TypeScript Docs

Docker（Day: マルチステージビルドで本番最適化）
  • 概要: build用とruntime用を分離して、軽量・安全なイメージを作る。
  • なぜ面白いか: pull時間・デプロイ速度・脆弱性面積を同時に改善できる。
  • 現場での使いどころ: Node/PHPアプリの本番コンテナ最適化。
  • ハマりどころ: .dockerignore未設定で不要ファイルまでコピーして肥大化。
  • 基礎の復習: レイヤーキャッシュを意識したCOPY順序の設計。
  • 今日の実践ミニ課題: 現行イメージとマルチステージ版のサイズ差を比較し、削減率を記録。
FROM node:20 AS build
WORKDIR /app
COPY . .
RUN npm ci && npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html

参考: Docker Multi-stage builds
