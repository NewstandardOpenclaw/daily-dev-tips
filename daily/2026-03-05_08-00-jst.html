Laravel（今日のサブトピック: Eloquent集約SQL / スコープ設計 / index最適化）
  • 概要: Eloquentで条件構築を整理し、重い集約はSQLへ寄せ、最後にindexで速度を作る回。
  • なぜ面白いか: アプリ層の書き方とDB性能が直結し、改善効果が数値で見える。
  • 現場での使いどころ: 管理画面の絞り込み一覧やダッシュボード集計APIの高速化に効く。
  • ハマりどころ: orWhere乱用で実行計画が崩れ、indexが効かないクエリになる。
  • 今日の実践ミニ課題: 検索API1本でwhen()化、EXPLAIN比較、複合index候補を1つ提案する。
  • 公式リンク: Eloquent / Query Builder / Database Indexes を合わせて読む。
$orders = Order::query()
  ->when($status, fn($q) => $q->where('status', $status))
  ->whereBetween('created_at', [$from, $to])
  ->with('user:id,name')
  ->paginate(30);
