Laravel（今日のサブトピック: Eloquent基礎 / N+1対策 / index設計）
  • 概要: Eloquentの取得設計を意識しつつ、N+1を抑え、DB indexで最終的な速度を作る日。
  • なぜ面白いか: 「コードの書き方」と「SQL性能」が直結して、改善結果が体感で分かる。
  • 現場での使いどころ: 一覧APIで遅いとき、with() と index の見直しで即効改善。
  • ハマりどころ: N+1を潰しても、where/sort列にindexがなければ遅いまま。
  • 基礎の復習: belongsTo/hasMany の向きと外部キー名は毎回確認。
  • 今日の実践ミニ課題: 既存一覧1本で「クエリ数」「実行時間」「EXPLAIN」をbefore/after比較。
// N+1対策 + 必要列だけ取得
$posts = Post::query()
  ->with(['user:id,name'])
  ->latest()
  ->paginate(20);

Laravelユニットテスト（Day 1: AAA + TestCase基礎）
  • 概要: arrange-act-assertの形で、まずは副作用の少ないサービス層からテスト。
  • なぜ面白いか: 壊れてもすぐ検知できる「安全ネット」ができる。
  • 現場ユースケース: 料金計算ロジックの仕様変更を安全に反映。
  • ハマりどころ: Feature Testで何でも確認しようとして実行が遅くなる。
  • 使い分けメモ: ドメイン計算=Unit、HTTP/認可/画面遷移=Feature。
  • 今日の実践ミニ課題: 1サービスに正常系2件・異常系1件を追加、CIで実行。

参考: Laravel Testing / PHPUnit

Vue（Vue2主軸: computed vs watch）
  • 概要: Vue2では「派生値=computed」「副作用=watch」を厳密に分けると保守性が上がる。
  • なぜ面白いか: 再描画や依存関係の理解が進むとバグが減る。
  • 現場での使いどころ: 入力フォームで表示加工はcomputed、API再取得はwatch。
  • ハマりどころ: watchで状態を書き換え続けて無限ループ化。
  • 今日の実践ミニ課題: 既存watchを1つcomputedへ置換して差分確認。
  • Vue3ならどう書くか: computed() と watch() をComposition APIで同じ責務分離にする。

参考: Vue2 Computed & Watch / Vue3 Computed

Docker（開発環境の再現性を上げる）
  • 概要: Dockerfileを「変更が少ない層→多い層」の順にするとビルドが速くなる。
  • なぜ面白いか: 環境差分バグが減り、オンボーディングも短縮。
  • 現場での使いどころ: PHP + Node混在プロジェクトの統一開発環境。
  • ハマりどころ: COPY . . を早い段で実行してキャッシュが毎回無効化。
  • 今日の実践ミニ課題: DockerfileのCOPY順を見直してビルド時間を計測。
  • 補足: 開発時はvolume、CIは再現性重視で固定イメージタグ。

PHP関数 / 配列ヘルパー（素のPHP vs Laravel）
  • 概要: array_map/filter/reduce を押さえつつ、Laravel Collectionへ橋渡し。
  • なぜ面白いか: データ加工が読みやすくなり、意図がコードに残る。
  • 現場での使いどころ: API整形、CSV変換、画面表示用フォーマット。
  • 比較: 素のPHPは軽量、Laravel版はチェーン可読性とユーティリティが強い。
// 素のPHP
$names = array_map(fn($u) => $u['name'], $users);

// Laravel Collection
$names = collect($users)->pluck('name')->all();
  • 今日の実践ミニ課題: 既存array_map処理1つをCollectionへ置換して可読性比較。

参考: PHP配列関数 / Laravel Collections

Python実用活用（Day 1: CSV/JSON加工の自動化）
  • 概要: 日常の手作業整形（CSV→JSONなど）を最小スクリプトで自動化。
  • なぜ面白いか: 10分作業の繰り返しをゼロにできる。
  • 実務ユースケース: 営業CSVをAPI投入用JSONへ変換して夜間バッチに渡す。
  • ハマりどころ: 文字コード・改行コード・空値の扱いでデータ欠損。
  • 今日の実践ミニ課題: 手元CSV1つをJSON化し、件数一致チェックを追加。
import csv, json
rows = []
with open('input.csv', newline='', encoding='utf-8') as f:
    for r in csv.DictReader(f):
        rows.append({'id': r['id'], 'name': r['name'].strip()})
with open('out.json', 'w', encoding='utf-8') as f:
    json.dump(rows, f, ensure_ascii=False, indent=2)
