Laravel（今日のサブトピック: Eloquent集約 + SQL最適化 / Queue再試行設計）
  • 概要: Eloquentで集計を安全に書きつつ、実クエリを見てIndexとEXPLAINで詰める日。
  • なぜ面白いか: 「読みやすいLaravelコード」と「速いSQL」を両立できると、実装の説得力が一段上がる。
  • 現場での使いどころ: ダッシュボード集計APIでwithCount()と複合Indexを合わせてP95を短縮。
  • ハマりどころ: Eloquent側だけ調整して満足し、ソート列のIndex不足で本番だけ遅い。
  • もう1テーマ: Queueのbackoffとtriesを明示し、外部API障害時の連鎖失敗を防ぐ。
  • ミニ課題: 既存1エンドポイントで「Eloquent変更前後 + EXPLAIN + レスポンスタイム」を比較して記録。
  • 公式リンク: Eloquent・Queue・DB Queryの3本を読み、現場コードに1つ反映する。
$reports = Order::query()
  ->selectRaw('user_id, count(*) as total')
  ->whereBetween('created_at', [$from, $to])
  ->groupBy('user_id')
  ->orderByDesc('total')
  ->limit(20)
  ->get();

参考: Eloquent / Query Builder / Queues

Laravelユニットテスト（境界値 + モック最小化）
  • 概要: Unitは純粋ロジックに集中し、外部I/Oは薄いFake層で切る。
  • なぜ面白いか: テストが速くなるほど、修正→実行→改善のサイクルが回る。
  • 現場での使いどころ: 請求計算・割引判定・締め処理など、仕様が変わりやすい箇所の防波堤。
  • ハマりどころ: Mockを増やしすぎて実装追従テストになり、リファクタ耐性が落ちる。
  • 観点: 正常系だけでなく「0件/境界日付/上限超え」の3点を必ず入れる。
  • ミニ課題: 既存Serviceに境界値テスト3件を追加し、失敗させてから通す。
  • 公式リンク: Laravel Testing + PHPUnit assertionsを再確認。

参考: Laravel Testing / PHPUnit

Vue（Vue2主軸: computed/watch整理 + Vue3移行の足場）
  • 概要: Vue2のOptions APIで責務分離し、将来のComposition API移行を楽にする。
  • なぜ面白いか: 今の保守性を上げながら、移行コストを先払いで下げられる。
  • 現場での使いどころ: フォーム入力の整形はcomputed、API再取得や副作用はwatchに限定。
  • ハマりどころ: deep watchを多用し、予期せぬ再描画とデバッグ困難を招く。
  • Vue3補足: ref/computed/watchに置き換える際、責務境界が明確だと移植が速い。
  • ミニ課題: Vue2コンポーネント1つでwatchを1件削減し、computedへ置換。
  • 公式リンク: Vue2 guideとVue3 essentialsを併読。

参考: Vue2 Computed/Watch / Vue3 Watchers

Docker（開発速度を落とさないイメージ設計）
  • 概要: マルチステージ + キャッシュ戦略で、開発とCIの両方を速く安定させる。
  • なぜ面白いか: 「動くけど遅い」環境を、誰でも同じ速度で回せるようにできる。
  • 現場での使いどころ: Node/PHP混在プロジェクトで依存更新時の再ビルド時間を短縮。
  • ハマりどころ: build contextが肥大化し、無関係ファイル変更でキャッシュ崩壊。
  • 観点: .dockerignore整備、依存インストール層の固定、runtimeイメージ最小化。
  • ミニ課題: 直近Dockerfileを1本見直し、ビルド時間をbefore/afterで計測。
  • 公式リンク: Build cacheとCompose docsを確認。

参考: Docker Build Cache / Docker Compose

  • 実務ポイント: 通知文を短文化し、次アクションを1行で明記する。
  • ミニ課題: 1つの定期タスクをcron化し、失敗時の再送条件を定義。
  • 公式リンク: OpenClaw docs / GitHub。

参考: OpenClaw Docs / OpenClaw GitHub
