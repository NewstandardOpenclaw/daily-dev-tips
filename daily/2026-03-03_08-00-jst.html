Laravel（今日のサブトピック: Eloquent絞り込みSQL / API Resource / トランザクション）
  • 概要: Eloquentのクエリ絞り込み精度を上げつつ、API Resourceで返却形を安定させ、更新系はトランザクションで守る3点を学ぶ日。
  • なぜ面白いか: 取得性能・レスポンス品質・整合性を同時に改善でき、現場で効果が見えやすい。
  • 現場での使いどころ: 管理画面の検索APIで条件増加に耐える設計や、複数テーブル更新の安全化に直結する。
  • ハマりどころ: 条件分岐をifで乱立させてSQLが複雑化し、N+1やindex未活用を見落としやすい。
  • 実務メモ: when()で条件クエリを整理し、重い集計はSQL実行計画を先に確認してから最適化する。
  • 今日の実践ミニ課題: 既存検索API1本で「条件追加→Resource整形→トランザクション適用可否」を棚卸しして改善案を作る。
$users = User::query()
  ->when($role, fn($q) => $q->where('role', $role))
  ->with('profile:id,user_id,company')
  ->paginate(20);
