Laravel（サブトピック2つ）

サブトピックA: Eloquent + SQL（N+1対策）
  1. 概要: 一覧APIの遅さは、N+1・列取りすぎ・index不足の複合で起きることが多い。
  2. なぜ面白いか: 1回の改善で全ユーザーの体感速度が上がる、費用対効果の高い改善領域。
  3. 現場での使いどころ: with() + select() + EXPLAIN確認をセット運用する。
  4. ハマりどころ: eager loadingしても、関連テーブルにindexがなく結局遅い。
  5. ミニ課題: 既存API1本で「クエリ数・応答時間・EXPLAIN」をbefore/after比較。
  6. 公式リンク: Eloquent eager loading
  7. 公式リンク: Query Builder
  8. 短いコード例:
$users = User::query()
  ->select(['id','name'])
  ->with(['profile:id,user_id,avatar'])
  ->paginate(20);
サブトピックB: Validation + FormRequest整理
  1. 概要: バリデーションをControllerから分離し、FormRequestに責務を寄せる。
  2. なぜ面白いか: 仕様変更時の修正ポイントが明確になり、レビューが速くなる。
  3. 現場での使いどころ: LPフォームの入力検証やAPIの入力整形を安定化できる。
  4. ハマりどころ: ルールが肥大化して「どの画面の仕様か」不明になる。
  5. ミニ課題: 既存エンドポイント1つをFormRequest化し、テストを1本追加する。
  6. 公式リンク: Validation
  7. 公式リンク: Controllers

Vue2（+Vue3補足）

$parent / $children依存を減らして ref + emit へ
  1. 概要: 子は入力UI、親は状態/バリデーションAPIという責務分離を守る。
  2. なぜ面白いか: コンポーネントの独立性が上がり、差し替えやテストがしやすい。
  3. 現場での使いどころ: LPフォームで入力部品（input/radio/select）を共通化しやすい。
  4. ハマりどころ: $children[0] の順序依存で将来壊れる。
  5. ミニ課題: 1画面で $parent 呼び出しを3箇所 $emit に置換。
  6. 公式リンク: Vue2 custom events
  7. Vue3補足: 同じ設計を emits 定義 + Composition APIでより明示化できる。

React Day

Day 3: stateを最小範囲に閉じる
  1. 概要: stateは必要な最小の親に置く。広げすぎると再レンダリング増加。
  2. なぜ面白いか: パフォーマンスと可読性を同時に改善できる。
  3. 現場での使いどころ: フォームやモーダルの局所状態管理で特に効く。
  4. ハマりどころ: 派生値までstate化して、不要なuseEffectが増える。
  5. ミニ課題: 1コンポーネント分のstateを下位へ移動して差分確認。
  6. 公式リンク: Choosing the State Structure
  7. 公式リンク: You Might Not Need an Effect

webpack / Vite / npm / yarn（dev-fast対策）

起動遅延を計測→切り分け→改善
  1. 概要: 遅延は依存解決・変換処理・watch I/O・source mapが主因になりやすい。
  2. なぜ面白いか: 体感待ち時間の短縮は毎日の開発体験を直接改善する。
  3. 現場での使いどころ: 大規模フロントで dev 起動/再ビルドを安定化できる。
  4. ハマりどころ: まず設定をいじって、計測がないまま悪化させる。
  5. ミニ課題: time yarn dev-fast を3回測定し、改善案を1つだけ適用して再測定。
  6. 公式リンク: Vite performance
  7. 公式リンク: Webpack build performance

AWS（基礎→実務）

IAM最小権限と監視の最低ライン
  1. 概要: まずIAM最小権限、次にCloudWatchで異常検知の基礎を固める。
  2. なぜ面白いか: セキュリティと運用保守の事故率を下げられる。
  3. 現場での使いどころ: API運用で5xx/Latency/CPUの3点監視を最小セットにする。
  4. ハマりどころ: 初期をAdmin権限で始めて、後から戻せなくなる。
  5. ミニ課題: 1ロールを最小権限に見直し、影響確認手順を記録。
  6. 公式リンク: IAM best practices
  7. 公式リンク: CloudWatch

Python実用活用

CSV→JSON変換の業務自動化
  1. 概要: 定型データ変換をPythonで自動化し、手作業の転記をなくす。
  2. なぜ面白いか: 小さい自動化が積み重なると工数削減が大きくなる。
  3. 現場での使いどころ: マーケデータやフォーム出力の前処理に直結する。
  4. ハマりどころ: 文字コード（UTF-8/Shift_JIS）で失敗しやすい。
  5. ミニ課題: 1つのCSVをJSON化し、件数一致チェックを追加。
