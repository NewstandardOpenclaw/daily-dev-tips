Laravelユニットテスト

Unitは純粋関数・ドメイン計算、FeatureはHTTP/DB/認可の境界確認に分ける。
なぜ面白い？ 失敗原因の切り分けが早くなり、レビューで仕様議論に集中できる。
現場の使いどころ: 金額計算、クーポン適用、状態遷移ルール。
ハマりどころ: 何でもFeatureに寄せて実行時間が肥大化する。
ミニ課題: 遅いテスト上位3件を特定し、1件をUnit化して実行時間比較。
公式: Laravel Testing / PHPUnit

test('fee is rounded down', fn() => $this->assertSame(120, $service->fee(129)));

Vue（Vue2中心 + Vue3補足）

Vue2ではcomputed中心で派生状態を作り、watchは副作用用途に限定すると安定する。
Vue3補足: composable化でロジック再利用し、refとreactiveを混在させすぎない。
なぜ面白い？ 同じ見た目でも状態の整理だけで不具合率がかなり下がる。
現場の使いどころ: 検索UI、モーダル、タブ切替、非同期フォーム。
ハマりどころ: Vue2のmixin増殖で依存が追えなくなる。
ミニ課題: 1コンポーネントのmixinをcomposable相当へ置換して責務を可視化。
公式: Vue2 Guide / Vue3 Guide

Docker

開発時はレイヤー順最適化（依存→ソース）で再ビルド時間を短縮する。
なぜ面白い？ チーム全員の待ち時間が即効で減る。
現場の使いどころ: CI一致、ローカル再現、オンボーディング。
ハマりどころ: bind mount権限とキャッシュ破棄タイミングの混乱。
ミニ課題: 既存Dockerfileでビルド時間を計測し、最適化前後を記録。
公式: Build cache / Compose

AWS（開発者向け基礎→実務）

基礎はIAM最小権限、CloudWatchログ、S3/RDSの責務理解から開始。
実務はALB+ECS/Fargate運用とParameter Store管理、アラート設計が要点。
なぜ面白い？ 構成の選択が開発速度・障害対応力・コストに直結する。
現場の使いどころ: API配備、バッチ、監視、障害時の一次切り分け。
ハマりどころ: CloudWatchメトリクスを見ずに感覚運用してしまう。
ミニ課題: 5xx/Latency/CPUの3アラームを1サービスに定義して通知確認。
公式: IAM / CloudWatch

React

描画ロジックと副作用を分離し、stateは最小化して導出値で持つのが基本。
なぜ面白い？ バグの温床になる再レンダリング連鎖を防げる。
現場の使いどころ: フォーム、検索、一覧、権限制御UI。
ハマりどころ: useEffectに業務ロジックを詰め込みすぎる。
ミニ課題: 1画面で不要なuseEffectを1つ削って挙動差分を確認。
公式: React Learn / You Might Not Need an Effect

現場の使いどころ: APIレスポンス、フォーム、イベントデータ。

HTML&CSS（必須基礎）

HTMLはセマンティック要素とlabel関連付けを徹底し、支援技術対応を強化。
CSSはレイヤー設計（base/components/utilities）で競合を抑える。
なぜ面白い？ 基礎を整えるほどJS依存が減り、実装が軽くなる。
現場の使いどころ: フォーム改善、レスポンシブ対応、崩れ修正。
ハマりどころ: 詳細度の暴走で小修正が大工事になる。
ミニ課題: 既存フォームにlabel/aria/focus-visibleを追加して操作検証。

フロントエンドビルド最適化（webpack/Vite/npm/yarn）

Zapier+n8n比較

Zapierは非エンジニアにも導入が速く、n8nは分岐・自己ホストで拡張しやすい。
なぜ面白い？ 「スピード優先」か「自由度優先」かを実運用で判断できる。
現場の使いどころ: 通知連携、CRM同期、承認フロー。
ハマりどころ: Zapierは課金増、n8nは運用責任増で負担が逆転しうる。
ミニ課題: 同一フローを両方で作り、工数/保守性/コストを比較。
公式: Zapier Help / n8n Docs

OpenClaw

Cronは「時刻固定配信」、Heartbeatは「まとめ確認」に分けると通知ノイズが減る。
なぜ面白い？ 自動化の精度だけでなく、運用体験そのものを改善できる。
現場の使いどころ: 朝レポート、定期ヘルスチェック、要点通知。
ハマりどころ: タスクを細かく分けすぎてメンテ不能になる。
ミニ課題: 既存定期処理をCron/Heartbeatに再分類して1つ統合。
公式: OpenClaw Docs / OpenClaw GitHub
